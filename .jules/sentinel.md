## 2025-02-14 - IDOR in Chat Operations
**Vulnerability:** Insecure Direct Object References (IDOR) on chat operations (`get_chat`, `delete_chat`, `rename_chat`, `clear_chat`, `chat`) allowed any user to access, view, modify, or delete any chat session if they knew or brute-forced the `chat_id`. The application did not verify if the chat belonged to the currently authenticated user session.
**Learning:** Even if `chat_id`s are UUIDs (which makes them hard to guess), relying solely on object ID unpredictability is insufficient for access control. All object-level operations must explicitly enforce authorization checks comparing the object's owner ID to the authenticated user's ID.
**Prevention:** Ensure that every created object (e.g., chat session) stores the creator's `session_id`. For every endpoint that performs CRUD operations on that object, explicitly verify that `session_id` matches `session.get('session_id')` before proceeding. Return a `403 Forbidden` if the check fails.
