## 2024-06-18 - SSRF Vulnerability in User-Configurable URLs
**Vulnerability:** Server-Side Request Forgery (SSRF) allowed via user-configurable API URLs (`AIDR_BASE_URL` and `ollama_url`). Users could specify malicious internal or metadata IPs (e.g., `169.254.169.254`, `10.x.x.x`), and the server would attempt to make HTTP requests to those addresses.
**Learning:** In architectures where the backend acts as a proxy or client to dynamically configured third-party/local APIs based on user settings, every URL must be strictly validated. Validating user-provided URLs in Flask settings is critical because they directly feed into internal API clients (like the AIDR SDK or OpenAI-compatible clients).
**Prevention:** Always validate and sanitize user-provided URLs before establishing outbound connections. Enforce URL schemes (HTTP/HTTPS) and resolve hostnames to block access to private, link-local, and loopback IPs (except when explicitly required, such as allowing localhost for local tools like Ollama). Implement fail-secure logic where DNS resolution failures deny the connection or fail open securely depending on context.
