## 2025-02-23 - Prevented SSRF Vulnerability in Ollama URL Configuration
**Vulnerability:** The application accepted a user-configurable `ollama_url` in the `/api/settings` route without proper validation, leading to a Server-Side Request Forgery (SSRF) risk. A malicious user could input an internal URL or a cloud provider's metadata IP address (like `169.254.169.254`) which the backend would then blindly query via the Ollama provider logic, potentially exposing sensitive internal data or resources.
**Learning:** Even internal configuration variables or settings intended for development/local use can become attack vectors if exposed to users. It's critical to treat configuration endpoints as untrusted inputs and enforce strict validation on any URL paths supplied by the user, particularly checking for restricted or private IP spaces and schemes.
**Prevention:** Always validate and parse URLs provided by the user using a robust library (like `urllib.parse`). Ensure the URL uses allowed schemes (like `http` or `https`), possesses a valid hostname, and actively block known metadata or loopback IP addresses (unless specifically intended, and even then with extreme caution) before using the URL in backend requests.
