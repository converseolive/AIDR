## 2025-02-14 - Fix IDOR and Authorization Bypass in Chat Sessions
**Vulnerability:** The `app.py` stored chat sessions globally in a dictionary without verifying user ownership. Endpoints like `/api/chats` and `/api/chat/<id>` allowed any user to list, read, modify, and delete any chat session if they had the chat ID, leading to a critical Insecure Direct Object Reference (IDOR) and authorization bypass vulnerability.
**Learning:** In a Flask application without an explicit user authentication system, session objects can still be used to uniquely identify users across requests. However, this relies on developers explicitly mapping domain objects (like chats) to those session identifiers. The previous implementation missed this mapping entirely.
**Prevention:** Always attach an owner identifier (like a `session_id` or `user_id`) to objects upon creation. When performing CRUD operations, always retrieve the object and verify that the current user's identifier matches the object's owner identifier before proceeding.
